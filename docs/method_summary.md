# 方法概要

## 1. 时间对齐

磁强计数据使用系统时间，北京时间。NovAtel 转出的 GPGGA 时间字段按 UTC 处理，再转换到北京时间：

```text
t_BJ = t_GPGGA_UTC + 8 h
```

当前 GPGGA 流程默认不减 18 s 闰秒。若后续使用 GPS week / seconds 或 NovAtel 原生日志头时间，需要单独处理 GPS time 到 UTC 的闰秒转换。

## 2. 沿轨一维坐标

所有有效 SPAN/GPGGA 坐标先转换为局部 East-North 平面坐标，再用 PCA 拟合共享轨道轴。4.14 与 5.13 共用同一条轨道轴和同一个物理 0 点：

```text
s = dot(p - p0, u)
```

其中 `p` 是局部平面坐标，`p0` 是轨道 0 点，`u` 是轨道方向单位向量。磁图输出间隔为 0.5 m。

## 3. 磁图构建

每个 0.5 m 地图点可能包含多趟观测。融合磁图默认使用鲁棒中位数：

```text
m_map(s) = median(m_1(s), m_2(s), ..., m_N(s))
```

同时保留均值、标准差和 MAD：

```text
MAD(s) = median(|m_i(s) - median(m_i(s))|)
```

中位数比均值更能抵抗掉头、尖峰和单趟局部干扰。

## 4. 三轴方向处理

小车往返时车体方向可能改变，磁传感器固定在车体上，因此 raw `x/y/z` 不能直接跨趟平均。当前处理逻辑是：

1. 每个连续观测段单独估计车体系磁场中位数；
2. 对 `x/y/z` 去除该段中位数，得到车体磁异常；
3. 根据沿轨距离增减判断运动方向；
4. 将车体磁异常映射到统一轨道坐标系，输出 `*_track_anom` 特征。

当前最稳定的跨日匹配特征仍是总场高通值。三轴特征可作为辅助候选，但不适合作为唯一主特征。

## 5. baseline 匹配

基础匹配使用滑窗归一化互相关 NCC。给定 query 曲线 `q` 和参考磁图窗口 `m_k`，分别标准化后计算：

```text
NCC(k) = mean(q'_i m'_{k+i})
```

得分最高的窗口位置作为预测位置。NCC 简单可解释，但容易被重复磁签名干扰。

## 6. HMM/Viterbi 匹配

HMM 将沿轨位置 `s_t` 作为隐藏状态，将磁强计观测 `z_t` 作为观测。目标是寻找最可能的位置轨迹：

```text
s_1, s_2, ..., s_T
```

观测似然来自磁场与磁图的相似程度，转移约束来自速度上限和运动方向。Viterbi 算法用于在时间-位置网格中寻找总得分最高的轨迹。

## 7. 当前最优 ProgressMarginSelector

当前最优方法不是单一路径匹配，而是候选选择：

1. 使用总场高通特征，分别以 `vmax = 1.0 / 1.2 / 1.4 m/s` 运行 HMM/Viterbi；
2. 生成一条轴校准 HMM 候选；
3. 对每条候选计算轨迹总进度；
4. 与 INSPVAX 速度积分得到的粗进度比较：

```text
progress_compat = |log((candidate_progress + 10) / (imu_progress + 10))|
```

5. 优先选择进度一致性好、Viterbi final score margin 较高的候选；
6. 只有当轴候选置信度和进度一致性都明显优于总场候选时才切换到轴候选。

SPAN/GPGGA 真值只用于最终评价，不参与候选选择。
